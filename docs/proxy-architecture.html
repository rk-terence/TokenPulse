Lifecycle & UI State
@MainActor
LocalProxyController
@MainActor · @Observable
Lifecycle owner for the entire proxy subsystem. Publishes observable state for SwiftUI binding. Owns startup/shutdown, UI refresh scheduling, and manual keepalive dispatch.
Lifecycle start(port:anthropicURL:openAIURL:) → initializes server, forwarders, logger. stop() → graceful teardown of all components.
UI Refresh 2-second periodic snapshots via startRefreshTask(). 50ms coalesced traffic refresh via scheduleTrafficRefresh(). Session expiration sweeps on each tick.
Observable State isRunning, listeningPort, proxyStatus, sessionActivities, manual keepalive in-flight / result flags, onTrafficEvent callback.
Network Layer
Sendable (lock-protected)
ProxyHTTPServer
Network.framework · NWListener
HTTP/1.1 listener bound to 127.0.0.1 (IPv4 loopback only). Parses request line + headers incrementally. Validates Content-Length, rejects duplicates (request smuggling prevention).
Constraints Max headers: 64 KB
Max body: 50 MB
POST only: /v1/messages or /v1/responses
Concurrency LockedBox<[NWConnection]> tracks connections.
CancellableTask wraps async handlers race-safe.
NWResponseWriter
ResponseWriter protocol
Serializes writes through dedicated DispatchQueue. Supports chunked transfer encoding (streaming) and buffered responses (non-streaming). Terminal chunk: 0\r\n\r\n.
Request Forwarding
Sendable (immutable config)
ProxyForwarder
Sendable · per-flavor instances
Routes requests to upstream APIs. Manages two distinct forwarding paths based on the stream field in the request body.
Streaming Path Per-request ephemeral URLSession + StreamingDelegate. Chunks forwarded at TCP segment size via writeChunk(). Body accumulated up to 4 MB for token parsing. withTaskCancellationHandler for disconnect detection.
Non-Streaming Path Shared nonStreamingSession for TCP/TLS reuse. NonStreamingPoolDelegate multiplexes by task ID. Response accumulated fully, written with Content-Length.
Error Handling: Invalid URL → 502 · Network error → 502 · Upstream 4xx/5xx → forwarded as-is · Client disconnect → logged

ProxySessionStore
actor
Session lifecycle, request activity tracking, token accumulation, and lineage evaluation.
Session State sessionID, lastSeenAt, in-flight count, token accumulators, keepalive mode, disable reason, lineage fingerprint. 24h session retention, 10m tracked-session visibility, 60s other-traffic visibility.
Lineage Tracking Phase 1: identify main agent from first tools-bearing request (2-request window). Phase 2: detect divergence (cache-invalidating changes) vs. continuation (append-only).
Request State Machine: .uploading → .waiting → .receiving → .done
ProxyMetricsStore
actor
Aggregated counters across all sessions. Global totals for requests, manual keepalives, cache hits, and token usage.
Savings Formula:
max(0, cacheReads*1.15 - keepalives*0.10)
Manual Keepalive
LocalProxyController.performManualKeepalive()
manual dispatch only · no background loops
Sends a single minimal Anthropic keepalive request on user action. Supported only for tracked Anthropic sessions whose lineage body + headers are established.
Request Build Builds stream=false + minimal max_tokens body from stored lineage. Reuses saved auth/version headers while skipping Host, Content-Length, Transfer-Encoding.
Behavior Uses URLSession.shared for a one-shot POST to /v1/messages. Success / failure is logged and counted, but no automatic loop or reconfigure path exists.
