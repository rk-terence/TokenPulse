Request Lifecycle

Complete data flow of an inbound HTTP request through the TokenPulse proxy

ProxyRequestState Machine

STEP 1
Connection Accept ProxyHTTPServer
NWListener accepts inbound TCP connection on 127.0.0.1:<port>. Connection registered in LockedBox<[ObjectIdentifier: NWConnection]>. Dispatched to processing queue.
STEP 2
Header Parsing & Validation ProxyHTTPServer
Reads data incrementally (up to 64 KB headers). Parses request line: method, path, HTTP version. Extracts all headers. Validates:
  • Method must be POST for supported routes
  • Path must be /v1/messages or /v1/responses
  • Rejects duplicate or invalid Content-Length values
  • Body read up to 50 MB when Content-Length is present
On Failure
  • Known route with non-POST method → 405 Method Not Allowed
  • Unknown route → 404 Not Found
  • Duplicate / invalid Content-Length or incomplete body → 400 Bad Request
STEP 3
API Flavor Routing ProxyHTTPServer
requestValidator callback identifies the ProxyAPIFlavor based on path. Creates ProxyHTTPRequest (method, path, headers, body). Creates NWResponseWriter for this connection. Wraps handler in CancellableTask for race-safe lifecycle.
/v1/messages
.anthropicMessages
Routed to Anthropic forwarder
/v1/responses
.openAIResponses
Routed to OpenAI forwarder
STEP 4
Session Identification ProxyForwarder
API handler derives the session key. Anthropic traffic uses X-Claude-Code-Session-Id when present and is stored as anthropic:<id>; OpenAI Responses traffic is grouped as "other". Call sessionStore.touch(sessionID) — creates session if new, updates lastSeenAt if existing. Increment inFlightRequestCount.
Side Effects
  • New session created in ProxySessionStore
  • Extract model from JSON body
  • Store request context only for tracked Anthropic sessions
  • Generate requestID (UUID)
STEP 5
Request Registration ProxySessionStore
startRequest(id:sessionID:model:promptDescriptor:isMainAgentShaped:)
Creates ProxyRequestActivity in .uploading state. Records bytesSent from request body size. Registers in activeRequests[UUID] map.
Side Effects
  • eventLogger.logRequestStarted() → SQLite row ID
  • metricsStore.recordForwarded()
  • Traffic callback fires → UI refresh
STEP 6
Build Upstream Request ProxyForwarder
Build URLRequest to upstream API. Copy all headers except hop-by-hop (Connection, Transfer-Encoding, Keep-Alive) plus Host and Content-Length. Set upstream URL from config. Determine streaming mode from body's "stream" field. State transitions to .waiting.
stream: true or false?
STEP 7A
Streaming Forward StreamingDelegate
Per-request ephemeral URLSession + StreamingDelegate. uploadTask.resume() sends body upstream. Response headers arrive via AsyncStream<Result<HTTPURLResponse, Error>>. Write response status + headers to client with Transfer-Encoding: chunked.
Chunk loop: For each upstream data chunk:
  • markFirstDataReceived(id:) on first chunk
  • updateRequestBytes(id:additionalBytes:) — bump cumulative + fire traffic callback
  • Accumulate response body (max 4 MB for token parsing)
  • writeChunk(chunk) to client
Terminal chunk 0\r\n\r\n sent after last upstream data.
Disconnect Handling
  • withTaskCancellationHandler detects client close
  • CancellationError logged, no response
STEP 7B
Non-Streaming Forward NonStreamingPoolDelegate
Shared nonStreamingSession for TCP/TLS connection reuse. NonStreamingPoolDelegate multiplexes callbacks by task.taskIdentifier. Per-task TaskContext registered for callback routing.
Response: Headers arrive → markRequestReceiving(id:). All response chunks accumulated in memory. Write full response with Content-Length to client.
STEP 8
Token Usage Parsing ProxyAPIHandler
parseTokenUsage(from:streaming:) extracts token counts from response.
Anthropic streaming: combine message_start input/cache usage with message_delta output usage.
OpenAI streaming: scan SSE for response.completed / response.incomplete usage payloads.
Non-streaming: parse the route-specific JSON usage object directly.
TokenUsage Fields
  • inputTokens, outputTokens
  • cacheReadInputTokens
  • cacheCreationInputTokens
  • inputTokensIncludeCacheReads flag
  • stopReason
STEP 9
Cost Computation ModelPricing
Look up per-model rates from ModelPricing table (input, output, cache_read, cache_write per million tokens). Compute estimated cost:
cost = input × input_rate
     + output × output_rate
     + cache_read × cache_read_rate
     + cache_create × cache_write_rate
Accumulation
  • sessionStore.recordTokenUsage() → per-session
  • metricsStore.recordTokenUsage() → global
STEP 10
Request Completion ProxySessionStore
markRequestDone(id:errored:tokenUsage:estimatedCost:)
State → .done. Move from activeRequests to doneRequestsBySession. Decrement inFlightRequestCount.
Prompt-based replacement: If new request's prompt descriptor contains an older request's descriptor (same model, messages are a superset), the older entry is replaced. Keeps only the most recent, longest context.
STEP 11
Lineage Evaluation ProxySessionStore
evaluateAndTrackLineage(body:headers:model:...)
Determines if this request continues the session's prompt cache lineage.
Phase 1 (first 2 reqs)
Identify main agent from first tools-bearing request. Window stays open until 2 requests seen. No main-agent by req 2 → disable manual keepalive for that session.
Phase 2 (subsequent)
Check: system prompt, messages (append-only?), model, tools, tool_choice, thinking. LineageFingerprint comparison.
Evaluation Results
  • .tracked → update lineage body + headers for later manual keepalive
  • .diverged → disable keepalive for the session and notify/log
  • .pendingIdentification → wait for more requests
  • .ignored / .alreadyDisabled → no keepalive state change
STEP 12
Event Logging ProxyEventLogger
logRequestCompleted(requestID:...) — UPDATE the row created in Step 5. Writes: status_code, duration_ms, upstream_request_id, token counts, cost estimate.
Optional
  • If saveProxyPayloads: INSERT into proxy_request_content
  • Prune check (24h retention, 5min sweep interval)
STEP 13
Status Snapshot ProxyEventLogger
writeStatusSnapshot() writes ~/.tokenpulse/proxy_status.json. Throttled to 1-second minimum interval. Pending writes coalesced — only the latest data is written when the timer fires. Atomic JSON write (write-then-rename).
STEP 14
Manual Keepalive Availability LocalProxyController
If lineage was .tracked, the session may become eligible for a later manual keepalive send from the popover. No extra network request is emitted here.
Eligible Later
Anthropic session with stored lineage body + headers. User can switch the row to .manual and later send a minimal /v1/messages keepalive via URLSession.shared. Result is recorded in session stats, metrics, and proxy_keepalives.
Not Eligible
• OpenAI Responses / other traffic
• No lineage established yet
• Keepalive disabled due to divergence or no main-agent candidate
COMPLETE
Response Delivered
Client has received the full response (chunked stream or buffered body). Connection closed. Request fully tracked across all subsystems. UI updated via coalesced traffic refresh (50ms debounce). Session lineage is now available for a later manual keepalive if eligible.
