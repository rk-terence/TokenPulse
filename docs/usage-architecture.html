Core Orchestration
ProviderManager
@MainActor · @Observable
Central orchestrator for provider lifecycle, refresh coordination, and state management. Manages ProviderState per provider with concurrency guards.
Refresh Race 30s timeout via withThrowingTaskGroup. Two racing tasks: fetch vs. sleep timer. First wins, other cancelled.
Generation Guard Each refresh gets unique refreshGeneration ID. Stale completions rejected on mismatch — prevents race conditions.
In-Flight Tracking inFlightRefreshes[String: Task]. Manual refresh cancels in-flight automatic. Prevents overlapping fetches per provider.

ConfigService
@Observable · Schema v6
Persistent config at ~/.tokenpulse/config.json.
Stores: enabledProviders, pollInterval, proxy port, Anthropic/OpenAI upstream URLs, keepalive UI settings, and logging flags. Auto-save on didSet. Auto-migration across schema versions.
